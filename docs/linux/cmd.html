文件权限 ​

第一个权限字母表示文件类型，第二个权限字母表示文件所有者的权限，第三个权限字母表示文件所在组的权限，第四个权限字母表示其他用户的权限。

对于目录而言，w 表示可以删除目录下的文件，x 表示可以进入该目录。

  • 文件类型

第一个字母|文件类型 -|- d|目录 l|符号链接 b|块设备文件 c|字符设备文件 s|套接字文件 p|管道文件 -|文件

  • 权限权值

权限|权值 -|- r|4 w|2 x|1

  • 常见权限

数字|权限 -|- 7|rwx 6|rw- 5|r-x 4|r-- 3|-wx 2|-w- 1|--x 0|---

shell
# 修改文件用户组
chgrp groupname filename

# 修改目录下所有文件用户组
chgrp -R groupname filename

# 修改文件所有者
chown username filename

# 修改目录下所有文件所有者
chown -R username filename

# 修改文件权限
chmod 777 filename
chmod a+x filename

# 修改目录下所有文件权限
chmod -R 777 filename
ad-note

    若目录用户具有权限 `rwx` , 目录下某个文件用户没有任何权限，则用户无法访问该文件，但可以删除该文件。
ad-tip
粘滞位权限`t`,只有创建文件的人才能删除，`w`权限无法删除，常用于`/tmp`目录下。

cut ​

cut 剪切字符串。

参数 -d 后的字符表示字段分隔符，默认为空格，表示在处理文件的内容时，将以其作为字段分隔符。

参数 -f 后的字符表示要提取的字段，默认为1，表示从第一列开始提取。

shell
cut -d ' ' -f2 file

ip ​

  • 创建了一个桥接设备 br0。
  • 将 eth1 接口加入到 br0。
  • 配置 br0 的 IP 地址为 10.0.0.2。
  • 启用 IP 转发功能。
sh
ip link add br0 type bridge
ip link set dev eth1 master br0
ifconfig br0 10.0.0.2
echo 1 >/proc/sys/net/ipv4/ip_forward
sh
ip route
sh
ip neighbor

iptables ​

  • 阻止所有输出到 8.8.8.8 的数据包。
  • 阻止所有来自 1.2.3.4 的输入数据包。
sh

iptables -I INPUT -s 1.2.3.4 -j DROP
